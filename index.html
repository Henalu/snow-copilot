What it does

Less guessing. More useful platform work.

Explain

Break down supported ServiceNow scripts with platform-aware guidance and clearer logic flow.

Ask

Refactor

Improve structure and implementation quality while staying closer to valid ServiceNow patterns.

Comment

Document

Generate Word documentation for scripts directly from the browser for change management.

Document UpdateSet

Capture visible Customer Updates and produce change documentation for sys_update_set records.

Public support scope

Only what is verified gets promised in public.

SN Assistant is intentionally specific about what counts as current public support and what is still evolving.

Business Rules & Scheduled Scripts

Verified support for server-side script contexts that need explanation, refactoring, commenting, and documentation.

$ sys_script / sysauto_script

Script Includes

Full support for Script Includes, including platform-aware guidance around object patterns and shared logic.

$ sys_script_include

Client Scripts, Fix Scripts, and UI Actions

Verified coverage for the supported interface-side scripting contexts already tested in the extension.

$ sys_script_client / sys_script_fix / sys_ui_action

Update Sets & Scripted REST

Document UpdateSet coverage includes visible Customer Updates, while Scripted REST Resources remain inside the verified support claim.

$ sys_update_set / sys_ws_operation
Under final validation

UI Scripts and Transform Scripts passed the current Chrome validation pass. They are next in line, but we are keeping them outside the formal public promise until Edge validation closes as well.

Still outside the current public promise

Background Scripts are not currently detected reliably enough for GA support. Unusual ServiceNow layout edge cases and heavily customized instances should also still be treated as outside the guaranteed scope.

Data and control

Multiple providers

Anthropic, OpenAI, Gemini, OpenRouter, custom endpoints, and local LLM setups are supported.

Local secret storage

Provider secrets such as API keys and custom auth headers stay in local browser storage on the current device.

Selective RAG

Breaking Trail retrieval happens locally inside the extension and is enabled only where it improves quality.
